兵棋特征数据字典：

一、 纯靠 class_type 与 domain 判定的机制 (无需填入 Capabilities)
这些单位在 JSON 中的 capabilities 应当保持为绝对干净的空数组 []。

防空保护伞 (AA Umbrella)

对应 class_type： spaag (自行高炮), sam_tac (战术防空), sam_stac (战略防空), inf_aa (防空步兵)。

系统作用： 战斗结算前，系统遍历防守方，只要发现这些 class_type，立刻激活 has_aa = True。此时，敌方 domain == "Air" / "Heli" 的单位遇敌权重恢复，不再无敌。结合你设计的 aa_area 属性，它们可以直接在网格上划定禁飞区，直接拦截固定翼飞机。

反炮兵与远程支援 (Indirect / Counter-Battery)

对应 class_type： spa_arty (自行火炮), towed_arty (牵引火炮)。

系统作用： 触发火炮对轰逻辑。如果双方都有此类 class_type，则原本安全的 domain == "Rear"（后方阵地）单位的遇敌权重提升，开始承受伤害池的伤害。

二、 必须使用 capabilities 的跨兵种标签
只有当一种能力横跨了多种截然不同的 class_type，或者需要对最终的战斗抛骰子结果（CRT 优势分）产生直接的加减修正时，才放入 capabilities。满打满算，只需要以下 5 个核心标签：

1. 装甲对抗体系 (Armor & AT)
这是冷战兵棋的灵魂。步战车和主战坦克都有装甲，武装直升机和步兵也都有反坦克导弹，这种复杂的交织必须用标签解耦。

HeavyArmor (重装甲)

适用单位： 先进mbt (如 M1A1, T-72m)。

系统作用： 防御端极高检定。如果对方方没有 HeavyAntiTank，则对方最终战斗优势分强制 -1.0（甚至无法造成有效伤害）。

Armor (常规装甲)

适用单位： 大部分 上一代 mbt (豹1系列), ifv (布雷德利), 部分spaag (猎豹)。

系统作用： 免疫轻武器。如果攻击方连 LightAntiTank 都没有，无法对其造成伤害，对方最终战斗优势分强制 -1.0。

HeavyAntiTank (重型反装甲)

适用单位： 挂载先进 ATGM 的单位(阿帕奇或TOW-2 班/运载车), 先进 mbt。

系统作用： 攻击端高阶检定。能够无惩罚击穿 HeavyArmor。

AntiTank（中型反装甲）

适用单位： 除先进 mbt 外的 mbt，部分ifv，挂载 ATGM 的单位（龙式反坦克导弹）。

系统作用： 能对付 Armor（常规装甲），遭遇 HeavyArmor 时仍受到战斗优势分 -0.5 的惩罚。

LightAntiTank (轻型反装甲)

适用单位： 所有的携带反坦克武器的步兵 infantry。

系统作用： 仅能对付 Armor（常规装甲），遭遇 HeavyArmor 时无效。

2. Recon (战术侦察)

适用单位： 骑兵战车（M3）、装甲侦察车（BRDM-2）、侦察直升机（OH-58）、精锐侦察狙击班。 
系统作用： 视野范围覆写。在回合开始或单位状态刷新时，如果营内没有任何存活的 Recon 实体，基础视野仅为 6 格（瞎子基线）。只要存在至少一个存活的 Recon 单位，整个营的视野范围提升至 10 格。 

3. Command (指挥与通信节点)

适用单位： 营部指挥车（M577）、指挥坦克、步兵指挥班。
系统作用： 系统通过 has_command_network() 函数遍历营内是否存活带有 Command 标签的单位。如果一方在战斗中的所有营均没有指挥单位，则会遭受巨额战斗优势分惩罚。
